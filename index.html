HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements work together to build structure of a webpage and provide a framework that makes it easy for a user to navigate a webpage.
  • HTML stands for Hyper Text Markup Language.
  • HTML is one of the three cornerstone languages used in web development.
  • HTML attributes are alawys defined in beginning element, never the closing element.
  • The link element connects an external resource to the HTML document.
  • The meta element defines matadata such as the character set, description, keywords, author, and viewport.
  • The style element defines the CSS style associated with a website.
  • The base element defines the external resource to the HTML.
  • When working with visible HTML elements, the content is contained between an opening and closing tag.
  • The br element creates a line of empty space, or a line break, between two blocks of elements.
  • The a element stands for anchor.
  • The a element creates a link to the same webpage or other webpages.
  • The href attribute points to the URL for the link.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS needs to be linked to HTML file for the code to be visible on website.
  • The link element creates the connection to the external file.
  • The rel attribute specifies the relationship between the current document and linked resource.
  • The href attribute specifies the location (URL) of the external resource.
  • There are three ways to style a webpage using CSS; inline CSS, CSS style sheet, and external CSS style sheet.
  • When a developer wants to style an element directly in the HTML file, they use inline CSS.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS style sheet allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • The box-shadow CSS property ads shadow effects around an elements frame.
  • Any HTML element assigned to the class '.card', will gain property assignments from the CSS.

Git

  • Git status: checks what branch we are currently on.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • Git commit -m to commit stagged files.
  • Git clone clones the remote repository into 2 places.
  • The git default branch is named main.
  • You can create a feature by branching off the main.
  • Git add can add a file directly as it looks to your next commit.
  • Git diff is whats changed but not stagged.
  • Git diff --staged is whats stagged but not yet committed.
  • Git merge is where you megrge specific branch history into a current one.
  • Git log shows all commits in the current branch history.
  • Git is a version control system that lets you manage and keep track of different versions of the project.
  • Git commands always start with 'git'.
  • Git commands are not the same as bash commands such as 'cd' and 'mkdir' .
  • Git is tracking changes locally on your own computer, but to do collaborating projects with others youll need to use a Git-based repository hosting service.
  • Git-based repository services are GitHub, Bitbucket, and SourceForge.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • For something to be a string it has to be surrounded by quotation marks.
  • A boolean is a true or false value. The words true and false are special keywords that dont need qoutes around them.
  • In javaScript language an array has an index as 0 instead of 1 which means the first array will always start with zero.
  • Commenting out code makes it inactive without deleting it.
  • Put /* before and */ after the block of code to comment out.
  • "let" keyword, followed by the name you give to a variable.
  • When finshed declaring variable, close it off with a semicolon to end.
  • JavaScript is case sensitive. If the same word is written with one in capital and the other lower case, they are not the same.
  • After declaring a variable you can give it a vaule.
    • After assigning a value to a variable, you can change it later in the code.
  • To signify that the value is a string, enclose it in single or double quote marks.
  • Numbers don't have quotes around them.
  • An array is a structure that allows you to store mulitiple values in asingle reference.
  • An object can be anything. Everything in JavaScript is an object and can be stored in varible.
  • Assignments (=) assigns a value to a variable.
  • Strict equality (===) performs a test to see if two vaules are equal. it returns a true/false (boolean) result.
